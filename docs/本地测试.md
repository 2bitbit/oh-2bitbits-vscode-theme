# 本地测试

## 修改主题颜色

主题颜色由 TypeScript 生成，**不要直接编辑 `themes/` 目录下的 JSON 文件**，修改会在下次生成时被覆盖。

| 文件 | 作用 |
|---|---|
| `generator/theme/colors.ts` | 基础色板 |
| `generator/theme/editor.ts` | 编辑器 UI 颜色（背景、边框、状态栏等） |
| `generator/theme/theme.ts` | 语法高亮 token |
| `generator/theme/semantic-colors.ts` | LSP 语义着色 |

## 方式一：热更新（推荐）

在终端启动 watch 模式，修改 `generator/` 下的文件后会自动重新生成主题 JSON：

```powershell 
npm run theme:watch
```

然后在 VS Code 中按 `F5` 启动调试实例。测试实例会禁用其他扩展并加载本主题。每次 `theme:watch` 重新生成后，在测试实例里执行 `Developer: Reload Window` 即可看到最新颜色。

## 方式二：手动生成后重启

```powershell
# 重新生成主题 JSON
npm run theme
```

然后在调试实例中执行 `Debug: Restart`。

## 本地安装 .vsix 测试

如果想在正式的 VS Code 环境（非调试实例）中测试：

```powershell
# 生成主题 JSON + 打包
npm run vscode:prepublish
npx vsce package --no-yarn
```

安装后在 VS Code 的颜色主题选择器（`Ctrl+K Ctrl+T`）中选择即可生效。
