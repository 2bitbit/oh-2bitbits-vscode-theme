# 发布新版本

## 1. 确认改动已生成

```powershell
npm run theme
```

检查 `themes/` 下的 JSON 是否已更新。

## 2. 更新版本号

```powershell
# 小修复（1.0.0 → 1.0.1）
npm version patch

# 新功能（1.0.0 → 1.1.0）
npm version minor

# 破坏性变更（1.0.0 → 2.0.0）
npm version major
```

该命令会自动修改 `package.json` 中的版本号，并创建一个带版本号的本地 git commit 和 tag（例如 `v1.0.1`）。

## 3. 更新 CHANGELOG

在 `CHANGELOG.md` 顶部追加本次版本的改动说明。

## 4. Push 触发自动发布

```powershell
git push origin main --follow-tags
```

GitHub Actions 检测到 `v*.*.*` 格式的 tag 后，自动执行：

1. 打包 `.vsix`
2. 创建 GitHub Release 并附上 `.vsix` 文件
3. 发布到 Visual Studio Marketplace
4. 发布到 Open VSX Registry

## 5. 验证

- GitHub Actions 状态：https://github.com/2bitbit/oh-2bitbits-vscode-theme/actions
- VS Marketplace：https://marketplace.visualstudio.com/items?itemName=2bitbit.oh-2bitbits-vscode-theme
- Open VSX：https://open-vsx.org/extension/2bitbit/oh-2bitbits-vscode-theme
